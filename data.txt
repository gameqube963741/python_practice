簡介
5 分鐘
假設您是一家公司的新軟體開發人員，該公司為商業客機撰寫航空軟體。 品質控制十分重要，且開發人員在使用 Git 進行版本控制的小組中工作。 您聽說過版本控制，但是您從未使用過 Git，所以您急於迎頭趕上！

您決定建置一個網站，讓您和您的朋友分享貓的照片，以便您可以在有趣的環境中學習 Git，然後再運用這些知識。 您開始使用 Git 來建置網站以追蹤變更，並在伺服器停機時，保留所有原始程式碼檔案的備份。

在此課程模組中，您將取得版本控制 Git 的簡介，以及如何自行使用 Git 從簡單錯誤中復原。 Git 一開始看起來有點難懂，有時甚至令人沮喪，但如果您逐步了解，就會發現它迅速成為世界上最受歡迎之版本控制系統的原因 — 它不僅適用於軟體開發人員，也適用於編寫文件並進行其他共同作業的團隊。

學習目標
在此課程模組中，您將會：

了解什麼是版本控制
了解分散式版本控制系統，例如 Git
建立新的 Git 專案並加以設定
使用 Git 進行及追蹤程式碼的變更
使用 Git 從簡單的錯誤中復原
